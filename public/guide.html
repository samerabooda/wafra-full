📞سير عمل مركز الاتصال
1
موظف CC يُدخل بيانات العميل
يُدخل رقم الحساب، الشهر، نوع الحساب، ويختار الفرع المستهدف. عمولته تُجلَب تلقائياً من ملف الموظف — يحددها المدير المالي مسبقاً لكل موظف.
2
CC يضغط "إرسال للفرع"
إشعار فوري يصل لمدير الفرع المعيّن. حالة الكرت تُصبح cc_pending. يظهر عدد الإشعارات غير المقروءة في شريط التنقل.
3
مدير الفرع يقبل أو يرفض
قبول: الكرت يظهر في قائمة "واردة من CC" جاهزاً للإكمال.
رفض: يكتب السبب — إشعار يصل لـ CC مع السبب. CC يمكنه إعادة الإرسال بعد التعديل.
4
مدير الفرع يُكمل بيانات الكرت
يختار البروكر (من موظفي فرعه)، يُدخل عمولة البروكر، المسوّق، الإيداع الأولي والشهري. عمولة موظف CC مُقفلة — لا يراها ولا يعدّلها.
5
الكرت يظهر عند CC والفرع معاً
بعد الإكمال يظهر في قائمة الفرعين. كل طرف يرى عمولته فقط. المدير المالي يرى الكل مع إجمالي العمولات ومقارنة بالحد $8.

📞Call Center Workflow
1
CC Agent enters client data
Enters account number, month, account type, and selects the target branch. Commission rate is auto-fetched from the employee profile — set by Finance Admin for each agent.
2
CC clicks "Send to Branch"
An instant notification reaches the target branch manager. Card status becomes cc_pending. Unread notification count appears in the navigation bar.
3
Branch Manager accepts or rejects
Accept: Card appears in "From CC" list, ready to complete.
Reject: Manager writes the reason — CC receives a notification with the reason. CC can edit and resend.
4
Branch Manager completes card data
Selects broker (from their branch employees), sets broker commission, marketer, initial and monthly deposits. CC agent commission is locked — branch cannot see or edit it.
5
Card visible to both CC and Branch
After completion, visible in both branches' lists. Each sees only their commission. Finance Admin sees everything including total commission vs the $8 cap.
